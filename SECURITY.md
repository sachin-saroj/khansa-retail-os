# Security Policy

## Supported Versions

Currently, the following versions are being supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| v1.0.x  | :white_check_mark: |
| < v1.0  | :x:                |

## Reporting a Vulnerability

Security is a high priority for Khansa Retail OS. If you discover a vulnerability, we would like to know about it so we can take steps to address it as quickly as possible.

Please do **NOT** report security vulnerabilities via public GitHub issues. Instead, please report them to the maintainers directly through private channels, or utilize the private security advisory feature on GitHub.

### Security Response Policy
- We will acknowledge receipt of your vulnerability report within 48 hours.
- We will send you regular updates about our progress toward a fix.
- Once the issue is resolved, we will publish a security advisory and acknowledge your contribution (unless you wish to remain anonymous).
- We kindly ask that you observe responsible disclosure principles and do not share the vulnerability publicly until a patch has been released.
